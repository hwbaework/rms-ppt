# 발전사업자(Generator) 화면 구성안 — 한 페이지 정리

> 에너지 자급자족 플랫폼 · 발전사업자 기준 GNB/LNB 및 각 화면 정의
> 근거: 유저저니 덱(rmsppt260605) + energy-frontend 실제 코드
> 수용가와 대칭 구조 · 정산을 독립 GNB로 분리(거래 ≠ 정산, 업계 표준)

---

## 발전사업자는 무엇이 제일 궁금한가

발전사업자(롯데SK에너루트·에스에너지·한일실리콘·울산미포ORC발전1호 등)는 전력을 생산해 공급한다.
자원유형에 따라 수익 모델이 갈린다 — 연료전지 SMP+2.2REC(≈239원/kWh·CHPS),
태양광 직접 PPA(140원/kWh)·자가소비, ORC 전력공급 SPC(≈100원/kWh·7,920h/년).
발전량이 곧 매출이라, 관심사는 두 축으로 압축된다.

1. 지금 잘 돌고 있나 — 설비 가동률·발전소 상태·발전량
2. 그래서 얼마 버나 — 수익·정산·미수금
3. 약정을 지키나 — 발전량 편차(초과/미달)·REC 발급
4. 계약은 안정적인가 — 잔여 계약 기간·갱신·해지 위약금
5. 시장가는 어떤가 — SMP·REC 시세

## 전체 메뉴 구조 (GNB → LNB)

```
GNB ① 대시보드      /dashboard             LNB 없음 · 단독 홈 · 허브
GNB ② 전력거래       "팔기" (자원 등록 포함)
        ├ 자원 관리        /generator/ppa/resources/register
        ├ 공급 신청        /generator/trading
        ├ 내 계약          /generator/ppa/contracts
        └ 전력거래시장(ETM) /generator/etm
GNB ③ 발전 모니터링   "운영 감시"
        ├ 발전 현황        /generator/ppa/dashboard
        ├ 발전소 모니터링   /monitoring/plant
        └ 이상감지         /monitoring/anomalies
GNB ④ 수익·정산      "내가 받는 돈"
        ├ 수익 분석        /generator/ppa/revenue/analytics
        ├ 정산 내역        /generator/ppa/revenue/invoices
        ├ 세금계산서       /generator/ppa/revenue/tax-invoice
        └ 발전량 편차      /generator/ppa/revenue/deviation
GNB ⑤ 이행·증빙      "REC·증빙"
        ├ REC·이행증빙     /ppa/documents/evidence
        ├ 보고서           /monitoring/reports
        └ 문서 보관함      /generator/ppa/documents
GNB 공통: 탄소거래 · e-Data · VPP · DT
```

핵심 설계: ① 자원 등록을 전력거래로 합침(공급의 전제조건·한 번 하는 셋업)
② 모니터링은 성격이 달라(상시 운영 감시) 별도 GNB
③ 수익·정산을 독립 GNB로(거래 ≠ 정산) — 수용가 '정산·요금'의 발전사판(받는 돈)

## GNB ① 대시보드 (/dashboard · LNB 없음 · 허브)

모든 위젯이 "요약 + 해당 GNB 딥링크" 2역할. 5개 GNB 전부가 진입 위젯을 하나씩 가진다.
관심사 순서(돌아가나 → 버나 → 지키나)대로 위에서 아래로 배치.

- 설비 가동률·현재 출력(현재출력÷용량 %) → ③ 발전소 모니터링 (이미 있음)
- 설비 상태 보드(정상·주의·점검·정지) → ③ 이상감지 (이미 있음)
- 발전량·발전시간(전일) + 오늘 시간대별 발전/소비 → ③ 발전 현황 (이미 있음)
- 금액(전일)·이번달 예상 수익 → ④ 수익 분석 (예상수익 추가)
- 미달 발전 수요기업(CFE<70%) → ④ 발전량 편차 (연결만 추가)
- 미수금·정산 알림 → ④ 정산 내역 (신규)
- 수요기업 비교·계약 갱신 알림 → ② 내 계약 (갱신 알림 신규)
- 자원 등록 유도 배너 → ② 자원 관리 (이미 있음)
- REC 발급 예정 → ⑤ REC·이행증빙 (신규)

→ 한 화면에서 5개 GNB 어디로든 한 클릭. 신규 추가는 미수금·계약 갱신·REC 발급 예정 3개.

## GNB ② 전력거래

**자원 관리** `/generator/ppa/resources/register`
신규 자원(발전소)·용량·자원유형(연료전지/태양광/ORC) 등록 / 현재 출력·오늘 발전량·상태·예상 REC 발급개수
→ 공급의 전제조건 · 거래 흐름의 입구

**공급 신청** `/generator/trading`
신청 프로세스 4단계: 신청 접수 → SPC 매칭 → 수요기업 승인 대기 → 매칭 완료 / 신청 취소
→ SPC 매칭 · ↔ 수용가 거래 신청과 매칭

**내 계약** `/generator/ppa/contracts`
총 계약 용량 · 잔여 계약 기간 · 중도 해지 위약금 / 등록 발전 자원 / 계약 변경·해지 신청
↔ 수용가 내 계약과 동일 데이터(상태 동기화) · → SPC·관리자 검토·승인

**전력거래시장(ETM)** `/generator/etm`
최신 SMP · 활성 계약 · REC 보유 · 총 거래 계약 / 시장 가격 추이(SMP/REC) / 내 거래 계약 목록
→ 시세 참고 (배치 위치는 검토 포인트)

## GNB ③ 발전 모니터링

**발전 현황** `/generator/ppa/dashboard`
현재 시각 발전량 · 오늘 누적 · 이번달 누적 · 이번달 예상 수익 / 발전량 추이 / 연도별 발전량 이력
→ 발전사의 일상 핵심 화면

**발전소 모니터링** `/monitoring/plant`
맵 기반 발전소 상세 상태(실시간)
※ 현재 발전사는 진입로 없음 — 신규 노출 필요

**이상감지** `/monitoring/anomalies`
이상 상황 관리 · 알림

---

## GNB ④ 수익·정산

**수익 분석** `/generator/ppa/revenue/analytics`
정산 일정 / 보고서·자동 발송 / 연도별 수익 추이

**정산 내역** `/generator/ppa/revenue/invoices` (1,068줄, 핵심 화면)
- 수익 항목: 전력량 대금(매출) · 부가정산금 · 망이용요금
- 차감 항목: 거래수수료(전력거래소) · 거래수수료(전력공급거래) · 관리 수수료
- 세금·기금: 부가세(+10%) · 전력산업기반기금 · 요금 조정
- 합계(VAT 포함) · 미수금(발행·미입금)
- → 수용가 지불액 − SPC 마진 = 발전사 수령액

**세금계산서** `/generator/ppa/revenue/tax-invoice`
발급번호 · 발급일 · 유형 · 공급받는자 · 사업자번호 · 발전소 · 공급량 · 단가 · 공급가액 · 부가세(10%) · 입금일 · 상태 · 입금계좌
→ 발전사는 발행·수금측 (수용가는 수취측)

**발전량 편차** `/generator/ppa/revenue/deviation` (화면 타이틀 "예상 발전량")
초과 발생 · 평균 SMP 시장가 · 예상 REC 발급개수 / 초과 발생 로그 / 매도 내역 PDF
→ 발전사는 초과 중심 (수용가는 부족 중심) · 동일 이벤트의 발전측 면

## 정산 데이터 흐름 (단일 데이터, 다중 뷰)

```
발전사 발전량(계량기/RTU)
        ▼
SPC 정산  ① 잠정 → ② 확정 → ③ 세금계산서 → ④ 수금/지급   (분쟁 시 재정산)
        ├─ 발전사 : 정산 내역(판매대금) = 수용가 지불 − SPC 마진
        ├─ 수용가 : 정산 내역(잠정/확정) → 납부
        └─ 관리자 : SPC 마진 = 청구액 − 지급액
```

발전량 편차는 동일 이벤트의 양면 — 발전사(초과) ↔ 수용가(부족) ↔ SPC(예측 vs 실측).

## 정리 — AS-IS 대비 무엇이 바뀌나

현재는 GNB 2개(대시보드 + 전력거래)뿐이고, 전력거래 LNB에 공급신청·내계약·발전현황·수익정산(단일)·자원관리가 혼재.

1. 모니터링 진입로 신설 — 저니 STEP2(/monitoring/plant)가 발전사 메뉴에 없음 → ③ 발전 모니터링
2. 정산 화면 LNB 노출 — 정산·세금계산서·편차 페이지는 있는데 '수익 분석'만 노출 → ④에 4개 모두
3. 대시보드 3중 중복 정리 — /dashboard·/generator/ppa/dashboard·/lease/dashboard → 홈 1개 + 발전 현황
4. 거래 ≠ 정산 분리 — 자원/거래/정산을 ②③④ 3개 GNB로 분리, 저니 STEP1~5 = GNB ②~⑤ 정렬
5. 자원 등록을 전력거래로 합침 — 공급의 전제조건

남은 결정: 모니터링 GNB 유지 vs 대시보드 흡수 · ETM 위치(②거래 vs 별도 시장) · 보고서 위치(⑤증빙 vs ③모니터링)
