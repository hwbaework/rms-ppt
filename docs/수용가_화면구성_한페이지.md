# 수용가(Consumer) 화면 구성안 — 한 페이지 정리

> 에너지 자급자족 플랫폼 · 수용가 기준 GNB/LNB 및 각 화면 정의
> 근거: 유저저니 덱(rmsppt260605) + energy-frontend 실제 코드

---

## 수용가는 무엇이 제일 궁금한가

산업단지 입주 수요기업(한국단조·무진네트워크·성림첨단산업 등)이 RE100 무료진단으로 진입해
발전원·계약유형을 고르고, 전력을 사고, 요금을 내고, 증빙을 받는다.
이들의 관심사 우선순위가 곧 화면 우선순위가 된다.

1. 얼마나 줄었나 — 한전 대비 이번 달/누적 절감액 (경영진 보고 숫자)
2. RE100 얼마나 달성했나 — 달성률·증빙 (수출기업은 CBAM·ESG 대응으로 시급)
3. 한전이었다면 얼마나 손해였나 — "한전 청구 vs 실제 PPA" 직접 비교
4. 이번 달 요금·다음 결제일 — 발전량 변동으로 예측 불안
5. 부족분 발생 여부 — 부족분은 비싼 한전 요금으로 보충됨

## 전체 메뉴 구조 (GNB → LNB)

```
GNB ① 대시보드      /consumer              LNB 없음 · 단독 홈
GNB ② 컨설팅
        ├ 무료 진단        /consulting/diagnosis
        ├ 마켓플레이스     /consulting/marketplace
        ├ 받은 제안        /consulting/proposals
        └ 컨설팅 현황      /consulting
GNB ③ 전력거래       "거래하기" (계약 + 사용)
        ├ 거래 신청        /ppa/trading
        ├ 내 계약          /ppa/contracts
        ├ 전력 현황        /ppa/status
        └ 사용량 분석      /consumer/usage
GNB ④ 정산·요금      "내가 내는 돈"
        ├ 정산 내역        /ppa/billing/settlement
        ├ 세금계산서       /ppa/billing/tax-invoice
        └ 사용량 편차      /ppa/billing/usage-deviation
GNB ⑤ RE100·이행     "증빙"
        ├ RE100 현황       /consumer/re100
        ├ 이행증빙         /ppa/documents/evidence
        └ 운영 보고서      /ppa/documents/report
GNB 공통: 탄소거래 · e-Data · VPP · DT
```

핵심 설계: ① 정산·요금을 전력거래에서 분리해 독립 GNB로 (거래 ≠ 정산, 업계 표준)
② 사용량 분석을 대시보드 → 전력거래로 이동(현황·분석 한 곳)
③ RE100·이행을 독립 GNB로 승격(2순위 관심사 상위 노출)

## GNB ① 대시보드 (/consumer · LNB 없음)

관심사 순서대로 위에서 아래로 배치. 모든 위젯은 상세 화면으로 딥링크.

- Zone 1 · 얼마나 아꼈나 — 이번달/누적 절감액·절감률 KPI + "한전 vs 실제 PPA" 월별 비교 차트(차이 음영) → "한전이었으면 +XXX만원 더 냈음"
- Zone 2 · RE100 달성 — 달성률 게이지·RE 비율·CO₂ 저감 + "증빙 발급" 바로가기
- Zone 3 · 이번 달 돈 관리 — 예상 요금·다음 결제일·미정산 + 부족분 경고(발생 시만)
- Zone 4 · 사용량 요약 — 총 사용량·수요 피크 + 미니 차트
- 제거: 발전량 추이 차트·발전소 상태 보드 (발전사 관심사)

## GNB ② 컨설팅

| LNB | 라우트 | 화면 내용 | 연계 |
|---|---|---|---|
| 무료 진단 | /consulting/diagnosis | 5단계 위저드(분야→기업현황→재생E목표→연락처→확인) | → 컨설턴트 신규 의뢰 |
| 마켓플레이스 | /consulting/marketplace | 컨설턴트 탐색·선택 | ← 컨설턴트 프로필 |
| 받은 제안 | /consulting/proposals | 제안 목록·상세(발전원·계약유형) | ← 컨설턴트 제안 발송 |
| 컨설팅 현황 | /consulting | 진행 중 컨설팅 추적·일정 (舊 컨설팅 홈) | ← 컨설턴트 프로젝트 관리 |

## GNB ③ 전력거래

**거래 신청** `/ppa/trading`
Lease PPA 문의 / 직접 PPA 유형 선택(Onsite·Offsite) / 거래 이력 / 신청 취소
→ SPC 매칭·중개 · ↔ 발전사 공급신청 · → 관리자 승인

**내 계약** `/ppa/contracts` (舊 '에너지 계약' 통합)
진행 5단계 추적: 신청접수 → SPC검토 → 거래소신고서 → 승인대기 → 효력발생
KPI: 총 계약 · 이달 비용 · CFE 매칭 · RE100 · 자원 믹스 / 갱신·변경·해지
↔ 발전사 동일 계약(상태 동기화) · → SPC·관리자 검토·승인

**전력 현황** `/ppa/status`
실시간 자원별 매칭 / RE100 누적 이행률 · 예상 정산금 / 할 일 · 최근 활동
← SPC 매칭 결과 · ← 발전사 공급량

**사용량 분석** `/consumer/usage` (대시보드에서 이동)
총 사용량 · 한전 사용량 · 평균 일사용량 · 수요 피크 / 시간대별·일별·월별 차트
→ SPC 발전량 예측 입력 · → 컨설턴트 진단 기초자료

---

## GNB ④ 정산·요금

**정산 내역** `/ppa/billing/settlement` (핵심 화면)
- KPI: 이달 총 비용 · 이달 순절감 · 다음 결제일 · 누적(YTD) · 미정산
- 정산 산출: 발전량(계량기/RTU) × SMP 단가 → 공급가액 / 24/7 매칭률
- 납부 통합: 청구 확인 → 납부 액션 (별도 화면 신설 안 함)
- 한전 비교 상세: 대시보드 Zone1의 상세판
- 상태: SPC 확정 전 '잠정' 배지
- ← SPC 정산 확정 · 납부=SPC 수금 · 발전사 판매대금과 차액=SPC 마진

**세금계산서** `/ppa/billing/tax-invoice`
- 수취 현황: 총 발급건수 · 총 수취 합계 · 미발급 · 결제 대기
- 상세: 발급번호 · 공급자 · 사업자번호 · 공급량 · 단가 · 공급가액 · 부가세 · 결제일 · 가상계좌
- 역할: 증빙·수취 확인 전용 (납부 액션은 정산 내역에만)
- ← SPC 발행 (확정 → 발행 → 수취 순서 강제)

**사용량 편차** `/ppa/billing/usage-deviation` (화면명 "사용량 부족 내역" → 라벨로 통일)
- KPI: 총 부족량 · 내부 보완률 · 정산 순영향
- 일별 추이 · 시간대별 분포 · 원인 분해 · 처리 분해 · 이벤트 로그 · 편차 정산 처리
- ← SPC 예측 vs 실측 · ↔ 발전사 편차(동일 이벤트 양면)

---

## GNB ⑤ RE100·이행

**RE100 현황** `/consumer/re100` (대시보드에서 이동)
현재 RE 비율 · 재생E 사용 · CO₂ 저감 / 월별 RE 비율 · 연도별 목표 vs 실적 · 에너지원 구성 · 로드맵
← 컨설턴트 RE100 인증 지원

**이행증빙** `/ppa/documents/evidence`
누적 발전·리스료·절감·증빙 제출 / 증빙 문서 + 미리보기·다운로드·이메일·공유
← SPC 거래 증빙 · ← 발전사 REC 발급(동일 라우트, 권한 분리 필요)

**운영 보고서** `/ppa/documents/report` (현재 74줄 · 보강 대상)
누적 발전량·리스료·한전 대비 절감·연간 추정 / 월별 비용 vs 절감 / 발급 이력
발행 주체(SPC/관리자) 미정의 → 보강 필요

---

## 정산 데이터 흐름 (단일 데이터, 다중 뷰)

```
발전사 발전량(계량기/RTU)
        ▼
SPC 정산  ① 잠정 → ② 확정 → ③ 세금계산서 → ④ 수금/지급   (분쟁 시 재정산)
        ├─ 수용가 : 정산 내역(잠정/확정) → 납부
        ├─ 발전사 : 판매대금 = 수용가 지불 − SPC 마진
        └─ 관리자 : SPC 마진 = 청구액 − 지급액
```

미러 라우트로 복제하지 않고 같은 데이터를 권한·관점만 다르게 표시.
정산 4상태가 전 페르소나 화면에서 동기화.

## 정리 — AS-IS 대비 무엇이 바뀌나

1. 대시보드 LNB 제거 → 단독 홈(절감·RE100 중심)
2. 에너지 계약·요금 내역 요약판 제거 → 내 계약·정산 내역으로 통합
3. 사용량 분석 → 전력거래로 이동
4. 정산·요금 → 독립 GNB 신설
5. RE100·이행 → 독립 GNB 신설
6. 컨설팅 홈 → 컨설팅 현황 명칭 변경
7. /ppa/billing ↔ /lease/billing 미러 폐기

남은 논점: 공통 GNB 4개 노출 여부 · 운영 보고서 발행 주체 · 이행증빙 권한 분리
