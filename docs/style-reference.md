# 발표 스타일 레퍼런스 — 에너지 모니터링 프로세스 맵 분석

> 원본: https://rms-ppt.rmsgroup.co.kr/energy-platform/monitoring-process
> **"참고"용이다 — 똑같이 복제하지 말 것.** 구조·문법은 가져오되 스킨은 우리 토큰으로,
> 글자는 더 크게(가독성 우선), 배경·장식은 자체 제작한다. (사용자 지시 2026-07-15)

## 1. 레퍼런스에서 가져올 구조 (문법)

| 요소 | 내용 |
|---|---|
| **vw 단위 비율 고정** | 모든 크기를 vw로 지정 → 창 크기가 바뀌어도 슬라이드 비율·배치가 항상 동일 |
| **표지** | 좌측 텍스트 블록(로고 → 태그라인 → 큰 제목 → 부제 → 날짜), 우측은 배경 그래픽 |
| **목차** | 상단 그래픽 밴드 + 하단 3×2 카드(로마숫자 Ⅰ~Ⅵ + 제목 + 한 줄 설명) |
| **본문 헤더** | 챕터 라벨(Ⅰ. …) + "파란 번호. 제목" |
| **거버닝 바** | 제목 바로 아래 한 줄 요약 박스 — 핵심 키워드만 파란 볼드 |
| **KPI 스탯 행** | 큰 수치(색상 다양) + 작은 회색 라벨, 4~5개 그리드 |
| **파이프라인** | "이름 + 부가설명" 노드를 화살표로 연결, 결론 노드는 진한 파랑 |
| **스윔레인** | 페르소나(참여자)별 행 — 라벨 열 + 색깔 노드 흐름. 시그니처 구성 |
| **정보 패널** | 흰 패널 + 제목 + 2~4열 미니카드(라벨 파랑 + 설명 회색) |
| **마무리** | 네이비 그라데이션 + 흰 "감사합니다" + 영문 태그라인 + rmsgroup.co.kr |
| **플레이어** | 상단 진행바 · 하단 blur 네비게이션(◀ 도트 n/N ▶) · 호버 시 풀스크린 버튼 |

## 2. 우리가 다르게 하는 것 (차별화 규칙)

1. **가독성 우선** — 레퍼런스 본문(0.65~0.82vw)은 너무 작다. 우리는 본문 최소 **0.78vw**,
   요약바 **0.95vw**, 설명은 말줄임 대신 **줄바꿈 허용**.
2. **팔레트는 저장소 토큰** — 레퍼런스의 `#0077EB/#022E6E` 대신 `blue-600(#2563EB)`,
   `blue-950(#172554)` 등 Tailwind 팔레트(CLAUDE.md §4)로 통일.
3. **배경 이미지 복사 금지** — 목차 밴드·본문 배경은 CSS(그라데이션 + 도트 그리드 + 링)로 자체 제작.
   표지는 `/images/Main.png`(16:9 교체 예정).
4. **본문 헤더는 칩(pill)** — 레퍼런스의 두꺼운 네이비 세로 바 대신 둥근 챕터 칩 + 얇은 헤어라인.
5. **여백·라운드 크게** — 카드 radius 12~14px, 패딩 넉넉히, 그림자는 은은하게.
6. **글보다 그림 우선** (사용자 지시 2026-07-15) — "도식"은 텍스트 카드 나열이 아니라
   **아이콘 + 곡선 화살표 + 라벨 필이 있는 진짜 그림**으로 그린다. 개념 설명 슬라이드는
   그림 다이어그램을 기본으로 하고 글은 보조로만. (예: `260715_carbon`의 `.dia` 메커니즘 다이어그램)
