에너지 자급자족 플랫폼 — 프로세스 맵

발전 자원 구축부터 RE100 이행까지, 6개 주체가 엮이는 생태계입니다. 흐름은 자원 유형·수익 모델·거래 방식에 따라 갈라집니다.

전체 프로세스 — 엔드투엔드 유저 저니

세로축 = 주체(레인), 가로축 = 6단계 여정. 각 칸은 그 주체가 그 단계에서 하는 행동·화면·분기입니다. 굵은 테두리=주 경로(happy path), ↘ 핸드오프=다음 주체로 넘김.
주 경로 (굵은 테두리 = 해당 레인 색) 해당 단계 미참여 분기발생 지점 ↘주체 간 핸드오프 추후 공통 기능 (현재 미적용)

자원별 수익 모델

① 현재 구조 — 장단점

페르소나마다 대시보드가 따로 있는 현재 멀티 대시보드 구조의 평가 (실제 페이지 분석 기준)

👍 장점

  • 페르소나 맥락 최적화 — 각 역할이 자기 업무에 필요한 위젯만 봄
  • 권한 경계 명확 — SPC 마진 등 민감정보를 관리자 전용 페이지에 격리
  • 장애 격리·독립 배포 — 한 페이지 변경이 다른 곳에 영향 적음
  • 역할별 딥링크 — URL로 특정 화면 직접 진입 가능

👎 단점 (중복이 핵심)

  • 위젯 중복 구현 — 발전량·수익·RE100·발전소 상태가 여러 대시보드에 각각 구현 → 유지보수 N배
  • 100% 복사본 존재 — /platform/lease/dashboard 는 /lease/dashboard 리다이렉트(완전 중복)
  • 데이터 정의 불일치 위험 — 같은 "이번달 발전량"이 페이지마다 다르게 계산될 수 있음
  • 탐색 혼란 — 발전사업자는 발전량을 3곳에서 봄 (어디가 정답?)
  • 라우트 트리 복제 — /lease/* ↔ /platform/lease/*, /generator/ppa/* ↔ /platform/ppa/*
